# The Variety of Data We Can Mine (slide8 - 7.6.2026)

While many people think of data as just simple tables or spreadsheets, data mining can actually be applied to a wide variety of complex and diverse information formats.

Data Mining ကို ရှုပ်ထွေးပြီး မတူကွဲပြားတဲ့ ပုံစံမျိုးစုံ (Complex and diverse information formats) တွေပေါ်မှာ အသုံးချနိုင်။

## 1. Complex and Connected Data

- **Structured Data and Graphs:** This includes information organized in specific formats or showing relationships between different points.

 သတ်မှတ်ထားတဲ့ ပုံစံစနစ်တကျ (Specific formats) တွေနဲ့ စုဖွဲ့ထားတဲ့ အချက်အလက်တွေ ဖြစ်သလို၊ မတူညီတဲ့ အချက်အလက် အမှတ်အသား (Data points) တစ်ခုနဲ့တစ်ခုကြားက ဆက်နွှယ်မှုတွေကို ပါ ပြသနေတဲ့ ဒေတာမျိုး ဖြစ်တယ်။ (Graphတည်ဆောက်ပုံမှာ Node တွေနဲ့ Link တွေ သုံးပြီး ချိတ်ဆက်ပြထားတဲ့ ပုံစံမျိုး)

- **Social Networks:** Mining the connections, groups, and interactions between people on platforms like Facebook or LinkedIn.

Facebook သို့မဟုတ် LinkedIn လိုမျိုး ပလက်ဖောင်းတွေပေါ်မှာရှိတဲ့ လူတွေအချင်းချင်းကြားက ချိတ်ဆက်မှုများ (Connections)၊ အုပ်စုဖွဲ့မှုများ (Groups) နဲ့ အပြန်အလှန် တုံ့ပြန်ဆောင်ရွက်မှုများ (Interactions) ကို Data Mining လုပ်ပြီး ရှာဖွေဖော်ထုတ်တာဖြစ်တယ်။ (ဥပမာ - ဘယ်သူတွေက ဘယ်သူတွေနဲ့ ပိုခင်လဲ၊ ဘယ်လိုလူတွေက အုပ်စုတစ်ခုတည်း ဖြစ်နေလဲ စတာတွေကို ရှာတာမျိုး)။

- **Multi-linked Data:** Data where many different records are connected to each other in a web of information.

အချက်အလက်တွေရဲ့ မှတ်တမ်း (Records) အမြောက်အမြားဟာ သတင်းအချက်အလက် ကွန်ရက်ကြီး (Web of information) တစ်ခုလိုမျိုး တစ်ခုနဲ့တစ်ခု ရိုးရိုးကြီးမဟုတ်ဘဲ နည်းလမ်းပေါင်းစုံနဲ့ အပြန်အလှန် ရစ်ပတ်ချိတ်ဆက်နေတဲ့ ဒေတာပုံစံမျိုး

## 2. Specialized and Challenging Databases

- **Object-Relational Databases:** More advanced databases that can store complex data types beyond simple text and numbers.

ရိုးရိုး Text(စာသား) နဲ့ Number(ဂဏန်း) တွေအပြင် ပိုမိုရှုပ်ထွေးတဲ့ ဒေတာအမျိုးအစား (Complex data types) တွေကိုပါ သိမ်းဆည်းနိုင်တဲ့ ပိုမိုအဆင့်မြင့်တဲ့ databaseစနစ် ဖြစ်တယ်။ (ဥပမာ - အသံဖိုင်တွေ၊ ဗီဒီယိုတွေ သို့မဟုတ် Custom Objects တွေကိုပါ တွဲလျက် ထည့်သွင်းသိမ်းဆည်းထားနိုင်တာမျိုး)

- **Heterogeneous and Legacy Databases:** 

**Heterogeneous:** Databases that contain many different kinds of data mixed together.

အမျိုးအစား မတူညီတဲ့ dataပုံစံပေါင်းစုံ (Different kinds of data) တစ်နေရာတည်းမှာ ရောနှောစုစည်း ပါဝင်နေတဲ့ databaseမျိုး ဖြစ်တယ်။ (ဥပမာ - စနစ်တစ်ခုတည်းမှာပဲ SQL tables တွေကော၊ NoSQL JSON documents တွေကော၊ ရိုးရိုး Text files တွေကော ပါဝင်နေတာမျိုး)

**Legacy database:** Older computer systems that might use outdated formats but still contain valuable historical information.

ခေတ်နောက်ကျနေတဲ့ ပုံစံစနစ်ဟောင်း (Outdated formats) တွေကို အသုံးပြုထားတဲ့ ကွန်ပျူတာစနစ်ဟောင်းတွေ ဖြစ်တယ်။ နည်းပညာအရ ခေတ်မမှီတော့ပေမဲ့ လုပ်ငန်းအတွက် အလွန်တန်ဖိုးရှိတဲ့ သမိုင်းဝင် အချက်အလက်ဟောင်း (Valuable historical information) တွေ ကျန်ရှိနေဆဲ ဖြစ်တယ်။

## 3. Multimedia and Text Information

- **Multimedia Databases:** Finding patterns in images, audio files, and videos.

စာသားသက်သက် မဟုတ်ဘဲ Images (ဓာတ်ပုံ/ပုံရိပ်များ)၊ Audio files (အသံဖိုင်များ) နဲ့ Videos (ဗီဒီယိုများ) ထဲကနေ စိတ်ဝင်စားဖွယ်ရာ ပုံစံတူညီချက် (Patterns) တွေကို ရှာဖွေတာမျိုး။ ဥပမာ - ဗီဒီယိုမှတ်တမ်းတွေထဲကနေ လူတွေရဲ့ မျက်နှာကို ခွဲခြားမှတ်မိစေတဲ့ စနစ်မျိုး (Facial Recognition)

- **Text Databases:** Analyzing "unstructured" data like emails, books, news articles, and reports.

ပုံသေသတ်မှတ်ချက် မရှိဘဲ ပုံစံမကျဘဲ ပြန့်ကျဲနေတဲ့ "Unstructured Data (ပုံစံမကျသော ဒေတာများ)" ကို ခွဲခြမ်းစိတ်ဖြာတာမျိုး။ ဥပမာ - ဝင်လာတဲ့ အီးမေးလ်တွေကို စက်ကနေ အလိုအလျောက် ဖတ်ပြီး Spam Mail ဟုတ်မဟုတ် ခွဲခြားတာမျိုး (Spam Filtering)

---

## 4. Location and Time-Based Data

- **Spatial and Spatiotemporal Data:** This involves mining information related to geographic locations (like maps) and how that data changes over time.

Spatial: ပထဝီဝင် တည်နေရာ (Geographic locations) တွေနဲ့ ပတ်သက်တဲ့ ဒေတာ

Spatiotemporal: အဆိုပါ တည်နေရာဒေတာတွေက အချိန်နဲ့အမျှ ဘယ်လိုပြောင်းလဲသွားလဲ (Changes over time) ဆိုတာ

Time-Series Data: အချိန်အပိုင်းအခြား တစ်ခုစီအလိုက် (ဥပမာ- တစ်စက္ကန့်ချင်း၊ တစ်ရက်ချင်း) ပုံမှန် တိုင်းတာမှတ်တမ်းတင်ထားတဲ့ ဒေတာတွဲများ။ (ဥပမာ: နေ့စဉ် စတော့ရှယ်ယာဈေးနှုန်း ပြောင်းလဲမှုများ၊)

Temporal Data: Time-series လို ပုံမှန်အချိန်အပိုင်းအခြားမျိုး မဟုတ်ဘဲ အချိန်ကာလ (Duration)၊ နေ့စွဲ (Date/Time) သို့မဟုတ် သမိုင်းဝင် အဖြစ်အပျက်တွေနဲ့ ဆက်စပ်ပြီး သိမ်းဆည်းထားတဲ့ ဒေတာမျိုး။ (ဥပမာ: လူနာတစ်ဦး၏ ဆေးရုံတက်ခဲ့သည့် ကာလမှတ်တမ်း၊ စာချုပ်သက်တမ်း ကာလများ)

Sequence Data: အချိန်ပေါ်တင် မဟုတ်ဘဲ အစီအစဉ်အလိုက် (Sequence) တစ်ခုပြီးတစ်ခု လာရမယ့် သတ်မှတ်ချက်ရှိတဲ့ ဒေတာပုံစံများ။ (ဥပမာ: ဝဘ်ဆိုက်တစ်ခုပေါ်မှာ အသုံးပြုသူတွေ တစ်ဆင့်ချင်းစီ နှိပ်သွားတဲ့ လမ်းကြောင်းမှတ်တမ်း)

Data Streams and Sensor Data: တစ်နေရာတည်းမှာ ရပ်မနေဘဲ Real-time တိုက်ရိုက် စဉ်ဆက်မပြတ် ဖတ်ရှု စီးဆင်းနေတဲ့ ဒေတာအမြောက်အမြား။ (ဥပမာ: IoT Sensors (အပူချိန်တိုင်း Sensor၊ မော်တာအမြန်နှုန်းတိုင်း Sensor) များမှ တက်လာသော ဒေတာများ၊ စမတ်ဖုန်းများမှ တိုက်ရိုက်ပေးပို့နေသော GPS Live Location စီးဆင်းမှုများ)

# Summary

Essentially, this slide tells us that data mining is extremely flexible. It doesn't matter if your information is an old office file, a satellite map, a collection of YouTube videos, or a network of friends; if you can store it digitally, you can likely mine it for useful patterns.
